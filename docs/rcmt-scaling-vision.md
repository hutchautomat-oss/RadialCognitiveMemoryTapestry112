## PHASE 2 — LANGUAGE (Next)

**What it is:** The .rcmt file format, the parser, the grammar derived from
geometry, and the translation engine. RCMT becomes a language that compiles
to TypeScript/Python/binary, rather than a tool written in TypeScript.

**What "done" looks like:**
- .rcmt binary file format is formally specified
- A parser reads .rcmt and outputs valid TypeScript, Python, or JSON
- A compiler reads TypeScript/Python and outputs .rcmt (reverse translation)
- An IDE extension (VS Code) renders .rcmt files with the color gradient
- The grammar is fully derived from geometry — no arbitrary syntax rules

**Why this matters:**
The parser is the wall that prevents all future drift. Once .rcmt is a
distinct parseable format, no other language or tool can accidentally
interpret RCMT color as executable instruction (Piet drift, etc.).
The format boundary IS the architectural guarantee.

**Products from this phase:**
- RCMT Dev: Codebase diagnostic tool (color gradient = epistemic health)
- RCMT Language: Open specification (TCP/IP model — standard is free)
- Licensed translation engines: TypeScript, Python, OpenAI memory API,
  LangChain documents, GraphQL schema
