To maintain a strict $\le 30\text{ms}$ execution budget—allowing the system to process intents and project geometry within one or two monitor refresh cycles—the pipeline utilizes several localized optimizations across the ingestion, compute, and rendering layers:

**1. Local Quantized Inference (Zero-Network Dependency)**
The system completely eliminates the latency of external API round-trips by running the semantic inference pipeline directly in the client's browser. It deploys an **8-bit integer quantized variant** (`model_quantized.onnx`) of the MiniLM model, which reduces the file size by 74% and keeps typical inference times around 28ms. 

**2. Off-Thread Hardware Acceleration (WebGPU/WASM)**
Text input is routed to a dedicated background Web Worker, ensuring the main UI thread is never blocked. This worker utilizes an automated initialization cascade: it **first attempts to allocate a WebGPU context** to execute high-density tensor math via massive SIMD parallelism and zero-copy GPU memory buffering. If WebGPU is unavailable, it gracefully falls back to a highly optimized, multi-threaded WebAssembly (WASM) execution environment utilizing 4 CPU cores. 

**3. The Cold-Start "Pre-Warm" Sequence**
Because WebGPU and WebAssembly engines suffer from a "cold start" delay during shader compilation, the pipeline executes a warm-up phase on initialization. During the application's splash screen, **a zero-filled dummy matrix is passed through the neural network**. This permanently compiles the WebGPU pipeline state objects and shaders upfront, guaranteeing the user's very first text entry will not hitch the UI.

**4. 1-Draw-Call VRAM Mutation**
To visually project the new node without overhead, the architecture completely bypasses React's virtual-DOM reconciliation. It utilizes a single `InstancedMesh` for all 8,000 maximum nodes and **mutates the underlying raw `Float32Array` attributes directly in the GPU VRAM**. This reduces what would be thousands of draw calls into exactly 1 draw call.

**5. $O(N \log N)$ Collision Spatial Indexing**
Before the render queue is updated, a collision engine must ensure the new node doesn't overlap with existing data. To prevent a brute-force $O(N^2)$ distance comparison from breaking the 30ms budget, the system **constructs a dynamic Bounding Volume Hierarchy (BVH)**. This spatial indexing calculates fast box-to-box intersections in $O(N \log N)$ time, resolving spatial conflicts instantly.
