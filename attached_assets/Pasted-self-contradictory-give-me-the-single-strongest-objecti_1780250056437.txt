WHAT IT IS RCMT (Radial Cognitive Memory Tapestry) is an optical-compression grounding substrate for AI agents. Instead of storing meaning as embeddings in a vector DB, it stores meaning as POSITIONS in a fixed 3D "foveated lattice" — 8,000 slots, ~224 KB, byte-stable on the wire. HOW MEANING IS STORED A node's position is a deterministic function of its slot index alone: one golden-angle Fibonacci spiral, radius = sqrt(index). Each memory is classified into one of five epistemic tiers — Fact > Scenario > Metric > Theory > Dream. Contiguous index ranges map to radial shells, so Facts sit dense at the core, Dreams sparse at the rim. The density gradient IS the encoded epistemology. THE WIRE (non-negotiable, each pinned by a test) - 28-byte packet: [slot u16][intent u16][x,y,z,scale f32][LWW timestamp f64] - No embedded peerId (server assigns identity over a JSON handshake) - No composite/vector clock — a single Float64 timestamp, strictly-greater wins - No embedding ever lives in the substrate, so it cannot drift on re-embedding THE GEOMETRY LAW (equally non-negotiable) Foveal Gradient Integrity: dense core, sparse rim must always read. Any placement/render change must preserve the gradient, never flatten to uniform density — because a VLM inherits the epistemic prior only if the densest visual signal coincides with the most-trusted memories. WHAT IT IS NOT - Not a vector DB, not a retraining loop. - Position does NOT encode semantic similarity or relationships — only tier + insertion order. Two nearby nodes are nearby in TIME, not in MEANING. - There are no edges / graph / "context map" between nodes. (A "comet" in the UI is just one node's motion trail as it migrates inward on reinforcement — not a link between memories.) THE CONSUMER The downstream reader is always a vision-language model doing FOVEAL scanning of the rendered lattice: dense high-confidence core first, sparse periphery as context. Claimed wins over text-RAG: optical cost scales with image resolution, not record count; and the artifact is byte-identical across model upgrades. NAVIGATION / UX (the human/dev view — just refined) The UI exists to demonstrate the substrate to evaluators. Two modes: - DRIVE mode: a distortion-free "scale dive" (Powers-of-Ten feel) into the dense core; constant FOV, no dolly-zoom warp, camera can never exit the sphere or strand in empty void. - WORK mode: cursor is a pure pointer (never drives the camera); selecting any intersection opens a per-slot code console for lawful read/write/promote. Crowding at the core is relieved ONLY by camera travel + perspective parallax + selective fading (focus-isolate dims neighbors on select; nearest cells fade to open a tunnel while diving) — NEVER by spreading nodes or flattening density. Snap-to-cell by click or by phrase. The felt goal: "you shrink as you approach," the view opens, each cell holds a readable apparent size. We also build RCMT BY the scientific method: every load-bearing claim is a falsifiable hypothesis with a kill condition. Architectural Gaps, Weaknesses, and Internal ContradictionsThe core tension in RCMT lies in forcing a Vision-Language Model (VLM) to act as a physical spatial processor while denying it the spatial structures it relies on to make sense of images.A. The Semantics-Time ContradictionThe Flaw: You state that position does not encode semantic similarity—only Epistemic Tier (radial shell) and Insertion Order (angle/distance within that shell). Two adjacent nodes are close in time, not meaning.The Contradiction: You also claim that the VLM handles grounding queries more accurately than text-RAG (H1). For a VLM to answer a complex grounding query from an image, related concepts must be grouped together spatially (spatial indexing). If a query requires correlating "Fact A" with "Fact B," but they were ingested six months apart, they will live on opposite sides of the core.The Consequence: The VLM's foveal glance cannot capture a semantic context window. It captures a temporal context window. RCMT unintentionally forces the VLM to use temporal proximity as a proxy for relevance, which fails standard RAG benchmarks.B. The Multi-Scale Text Legibility ParadoxThe Flaw: In the UX section, you state that when diving into the core, "each cell holds a readable apparent size."The Contradiction: If the core houses thousands of highly dense facts, and the camera is positioned to see the whole core, the text inside those cells must shrink below the sub-pixel rendering threshold of the rendered image. If the camera zooms in so that a cell is large enough to read, the rest of the core moves out of the VLM's field of view (FOV).The Consequence: A VLM cannot perform a "foveal scan of the whole dense core" if the text inside the core requires a high-resolution "scale dive" to become legible. If the VLM must look at 50 different zipped image crops to read the text, the claimed optical token-budget savings over text-RAG vanish.C. The Coordinate System IllusionThe Flaw: The 28-byte packet passes [x,y,z,scale f32] over the wire.The Contradiction: You state that a node's position is a deterministic function of its slot index alone via a Fibonacci spiral. If position is purely deterministic based on index, passing explicit x,y,z coordinates over the wire is redundant and introduces a vector for state drift. If a client alters the x,y,z manually, it violates the Geometry Law.2. The Single Strongest ObjectionVLMs do not possess native foveated spatial reasoning; they process images via coarse, uniform patch tokenization (ViT grids).Modern VLMs do not scan an image like a human eye moving a fovea. They cut an image into a rigid grid of patches (e.g., \(14\times14\) pixels per token) and flatten them into a sequence.When a VLM looks at your dense core, a single patch token will swallow dozens of microscopic, overlapping text nodes. The attention heads will blend these distinct strings into an unrecoverable semantic soup.You are designing a substrate optimized for human ocular mechanics and treating it as an optimization for transformer architectures that fundamentally lack those mechanics.3.
